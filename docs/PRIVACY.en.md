# Privacy and Safety

## What the open-source foundation stores

By default, paper-label stores the following in browser storage:

- metadata for papers you save;
- ranking datasets you import locally;
- folders, categories, tags, and notes;
- language, badge color, and badge size preferences.

The foundation does not automatically upload this data to a paper-label server. This repository does not include accounts, cloud sync, translation, or shared ranking services.

## Page access

The extension reads the current scholarly page to identify title, authors, DOI, journal, and structured metadata, then places badges and controls on that page. It should not read unrelated passwords, payment details, or form contents.

## Full-text actions

Full-text actions only open lawful search or access points. They do not bypass paywalls, break access controls, or download restricted content on your behalf.

## Your control

You can delete library records and export your data. Export any data you want to keep before uninstalling the extension.

## Reporting a security issue

Do not post cookies, access tokens, institutional credentials, restricted papers, or personal data in a public issue. Follow [SECURITY.md](../SECURITY.md) for security reports.
